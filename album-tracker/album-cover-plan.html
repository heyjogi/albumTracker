변경된 방향 요약

제거 CreatPurchase 커버 선택
구매 등록 시 커버 선택 없음. 등록 flow 그대로 유지.
추가 AllPurchases 인라인 편집
구매 목록에서 각 아이템에 커버 선택 UI 추가, 바로 업데이트.

DB 변경 — 최소 필요

1
수정 purchases 컬럼 2개 추가
cover_label text — 'A'/'B'/'C'/'D'/null
album_type text — 'POCAALBUM'/'INVENTORY'/'ID_PASS'/null (스냅샷)
2
확인 safe_store_albums에 album_type_id 노출
purchases → album_id → store_albums.album_type_id 경로로 타입 조회 가능. safe_store_albums view에 album_type_id가 없으면 view 수정 필요. 있으면 그대로 사용.
3
생략 store_album_covers 테이블
커버 이미지 없이 이름만 쓰므로 별도 테이블 불필요. A/B/C/D 라벨은 고정값으로 프론트에서 하드코딩.
album_type을 purchases에 텍스트 스냅샷으로 저장하면 — 기존의 store_name, album_name 방식과 동일하고 join 없이 AllPurchases에서 바로 탭 필터링 가능.

구현 순서

1
Supabase — purchases 컬럼 추가
cover_label text, album_type text 2개. safe_store_albums에 album_type_id 노출 여부 확인 (없으면 view 수정)
2
PurchaseItem — 커버 칩 + 인라인 업데이트
카드에 A/B/C/D/미정 버튼 추가. 탭 시 supabase.update({cover_label}) 즉시 호출
3
CoverMatrix 컴포넌트 작성
purchases 배열 받아서 스토어×멤버 매트릭스 + 앨범멤버×팀멤버 수량 테이블 렌더링
4
AllPurchases — 탭 + CoverMatrix 통합
POCAALBUM/INVENTORY/ID PASS/전체 탭 → album_type으로 필터 → CoverMatrix + PurchaseList 순서로 배치
5
기존 데이터 소급 입력
Step 2에서 만든 커버 칩으로 기존 구매건 직접 탭해서 입력. album_type은 album_id → store_albums.album_type_id 에서 조회해 자동 채우기 가능
